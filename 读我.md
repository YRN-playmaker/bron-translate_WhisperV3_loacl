# BroN-translate · Local 版部署教程（RTX 50 系显卡专用）

> 这是 [BroN-translate](https://bron-translate.org) 的本地开源版部署指南。如果你
> 只想直接使用、不想折腾环境，请用网页版 **bron-translate.org**。本仓库面向希望
> 在自己 GPU 上本地运行 Whisper large-v3 的用户。
>
> 本项目以 **AGPL-3.0** 许可证发布，商业/闭源使用请见文末「许可证」一节。

---

## 已验证测试环境

* Windows 11
* Python 3.11
* RTX 5060 Laptop（Blackwell，sm_120）
* CUDA Driver 591.86 / CUDA Version 13.1（驱动显示值）
* faster-whisper 1.1+
* **ctranslate2 4.8.0**
* **nvidia-cublas-cu12 12.4.5.8 + nvidia-cudnn-cu12 9.1.0.70**（关键：见第 6 步）

> ⚠️ RTX 50 系（Blackwell）是新架构，推理库仍在适配期。本教程记录的是一套
> **实测可跑通**的版本组合，强烈建议照抄版本号，不要随意升级 cuBLAS/cuDNN。

---

## 1. 克隆项目

```bash
git clone https://github.com/YRN-playmaker/bron-translate-local.git
cd bron-translate-local
```

---

## 2. 创建虚拟环境

```cmd
python -m venv .venv
.venv\Scripts\activate
```

看到命令行前缀变成 `(.venv)` 即进入成功。

---

## 3. 安装依赖

```cmd
pip install -r requirements.txt
```

---

## 4. 配置 DeepSeek

```cmd
copy .env.example .env
```

编辑 `.env`，填入你的 DeepSeek API Key：

```env
DEEPSEEK_API_KEY=你的KEY
```

（没有 Key 时转写仍可运行，但翻译会被禁用。）

---

## 5. （可选）把 Whisper 模型下载到本地

默认首次转写会自动联网下载 large-v3（几个 GB）。若想避免每次联网或固定路径，
可手动下载到 `model/` 下，并修改 `.env`：

```env
WHISPER_MODEL=./model/whisper-large-v3
```

---

## 6. RTX 50 系必须安装并锁定 NVIDIA Runtime（关键步骤）

只装 faster-whisper 不够 —— GPU 推理需要 cuBLAS 和 cuDNN 运行时，而它们不会被
自动安装。**而且在 Blackwell 上，版本必须锁定**，否则会出现 cuBLAS 加载失败、
`CUBLAS_STATUS_NOT_SUPPORTED` 等问题：

```cmd
pip install nvidia-cublas-cu12==12.4.5.8
pip install nvidia-cudnn-cu12==9.1.0.70
```

> 为什么锁这两个版本：最新版 cuBLAS（如 12.9.x）/cuDNN 与 ctranslate2 4.8.0 在
> Blackwell 上组合会加载失败。`cublas 12.4.5.8 + cudnn 9.1.0.70` 是实测可跑通的
> 组合。

---

## 7. RTX 50 系 DLL 自动加载修复（已内置）

本项目的 `main.py` 顶部已内置 Windows DLL 路径自动注册逻辑
（`_register_nvidia_dll_dirs()`），会自动定位 venv 中的 NVIDIA DLL，避免
`cublas64_12.dll not found`。**你无需手动添加**，此处仅说明原理：

```python
def _register_nvidia_dll_dirs():
    if sys.platform != "win32":
        return
    import importlib.util
    spec = importlib.util.find_spec("nvidia")
    if not spec or not spec.submodule_search_locations:
        return
    root = list(spec.submodule_search_locations)[0]
    for sub in ("cublas", "cudnn", "cuda_nvrtc", "cuda_runtime"):
        path = os.path.join(root, sub, "bin")
        if os.path.isdir(path):
            try:
                os.add_dll_directory(path)
            except Exception:
                pass
            os.environ["PATH"] = path + os.pathsep + os.environ.get("PATH", "")
```

## 8. 验证 CUDA 链路

项目内已提供 `test_gpu.py`（用 tiny 模型秒级验证，不下大模型）：

```cmd
python test_gpu.py
```

它会先 `import main` 触发上面的 DLL 修复，再在 GPU 上加载 tiny 模型。出现：

```text
GPU OK
```

即表示 CUDA / cuBLAS / cuDNN 链路正常。

---

## 9. RTX 50 系推荐 .env 配置

```env
WHISPER_DEVICE=cuda
WHISPER_COMPUTE_TYPE=float16
```

**不要使用 int8：**

```env
# ❌ Blackwell 上 int8 系列（int8 / int8_float16 ...）当前会报
#    CUBLAS_STATUS_NOT_SUPPORTED，无法运行
WHISPER_COMPUTE_TYPE=int8
```

显存不太紧张时可改用 `float32`（更稳但更占显存），不要退回 int8。

---

## 10. 启动

```cmd
python main.py
```

或双击 / 运行：

```cmd
start.bat
```

启动成功会看到：

```text
[*] DeepSeek client ready.
[*] BroN-translate (Local) running at http://127.0.0.1:8000/
```

浏览器会自动打开 http://127.0.0.1:8000 。

## 常见错误

### cublas64_12.dll not found / cannot be loaded
1. 确认已按第 6 步**锁定版本**安装 cuBLAS/cuDNN；
2. 确认 `main.py` 的 DLL 自动加载逻辑存在（第 7 步，已内置）；
3. 注意 “not found” 与 “cannot be loaded” 不同：后者多为版本不匹配，按第 6 步
   锁版本即可解决。

### Whisper Ready 后无法识别
依次检查：浏览器麦克风权限、WebSocket 是否连上、VAD 是否触发、Whisper 是否真正
开始转录。可在 `flush_segment()` 内临时加 `print("flush triggered")` 辅助排查。

### float16 不工作 / tiny 能跑但 large-v3 失败
先跑 `python test_gpu.py`。若 tiny 通过而 large-v3 失败，通常是**显存不足**或
**模型路径错误**，而非显卡不支持 FP16。关掉占显存的程序（浏览器、Teams 等）再试。
